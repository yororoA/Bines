# 模块管理控制台使用说明

## 功能概述

模块管理控制台提供了一个基于 Web 的界面来管理和监控所有系统模块，无需打开多个终端窗口。

## 主要功能

1. **统一管理界面**：在一个 Web 页面中查看和管理所有模块
2. **实时日志显示**：每个模块的日志实时显示在页面上
3. **服务控制**：可以单独启动/停止某个服务，或一键启动/停止所有服务
4. **状态监控**：实时显示每个服务的运行状态（运行中/已停止）

## 使用方法

### 方式一：使用 Web 模式（推荐）

```bash
# 在项目根目录下运行
python start_modules.py --web
```

或者直接运行模块管理服务器：

```bash
cd server
python module_manager.py
```

然后访问：http://localhost:5000

### 方式二：使用传统控制台模式

```bash
python start_modules.py --console
```

## Web 界面功能

### 左侧服务列表
- 显示所有可用的服务
- 绿色圆点表示服务正在运行
- 红色圆点表示服务已停止
- 点击服务名称可以查看该服务的日志

### 右侧内容区域
- **顶部操作栏**：显示当前选中的服务名称，以及启动/停止按钮
- **日志显示区**：显示选中服务的实时日志输出
  - 错误信息显示为红色
  - 警告信息显示为黄色
  - 信息显示为青色

### 顶部操作栏
- **启动所有服务**：一键启动所有模块
- **停止所有服务**：一键停止所有模块

## API 接口

### 获取所有服务列表
```
GET /api/services
```

### 启动服务
```
POST /api/services/<服务名称>/start
```

### 停止服务
```
POST /api/services/<服务名称>/stop
```

### 获取服务日志
```
GET /api/services/<服务名称>/logs?max_lines=500
```

### 获取服务状态
```
GET /api/services/<服务名称>/status
```

### 启动所有服务
```
POST /api/services/start-all
```

### 停止所有服务
```
POST /api/services/stop-all
```

## 依赖安装

确保已安装以下 Python 包：

```bash
pip install Flask flask-cors
```

或者使用 requirements.txt：

```bash
pip install -r server/requirements.txt
```

## 注意事项

1. Web 管理服务器默认运行在端口 5000，确保该端口未被占用
2. 所有模块在后台运行，不会显示终端窗口
3. 日志会自动滚动显示最新内容
4. 服务状态每 2 秒自动更新一次
5. 日志内容每 1 秒自动更新一次

## 故障排除

### 无法访问 Web 界面
- 检查端口 5000 是否被占用
- 检查防火墙设置
- 查看控制台错误信息

### 服务无法启动
- 检查 Python 解释器路径是否正确
- 检查脚本路径是否存在
- 查看日志中的错误信息

### 日志不显示
- 检查服务是否正在运行
- 查看浏览器控制台是否有错误
- 确认服务有输出内容
